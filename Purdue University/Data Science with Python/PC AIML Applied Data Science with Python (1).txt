2) Primary topic and learning objectives

3) Key conceptual points taught

- Relationship to Series:
  - Series are single-column structures; multiple Series combined form a DataFrame.
  - Indexing is critical for alignment: values with the same index correspond across columns.

- Creating DataFrames from dictionaries:
  - A Python dictionary with keys mapping to lists can be converted to a DataFrame using pd.DataFrame(dictionary).
  - Dictionary keys automatically become column headers; lists become column values aligned by index.
  - Example discussed: columns like name, age, salary; 3 rows.
  - Instructor recommended starting each exercise in a fresh blank notebook.

- Creating DataFrames from lists:
  - When providing separate lists as data input to pd.DataFrame, column headers must be provided explicitly (they are not auto-generated from keys as with dictionaries).
  - Students were shown examples where multiple lists (list1, list2, list3, etc.) are passed together with a separate columns list (e.g., ["name","age","salary"]) to create a DataFrame.

- Creating DataFrames from NumPy arrays:
  - Workflow: start from Python lists, convert to a NumPy array via np.array([list1, list2, ...]) after importing NumPy as np, then pass the array to pd.DataFrame with explicit column headers.
  - Instructor reiterated that NumPy is a different package and must be imported prior to using its array() method.
  - Examples included creating arrays from 2 or 3 lists and then making DataFrames from those arrays.

- Imports and placement:
  - Best practice: place import statements at the top of the notebook, but the essential requirement is that imports must precede any code that uses them (e.g., import pandas and import numpy before using pd or np).
