## Key Concepts Covered

### 1. Missing Data: identification and high-level workflow
- First step: identify missing values (nulls) in the dataset (geetika emphasized using isnull / null-check methods).
- Second step: replace missing values with estimated values appropriate to the data type (categorical vs. numerical).
- Emphasis: treat categorical and numerical columns differently; choose strategy based on column type and distribution.

### 2. Strategies for Numerical Data (columns indexed 1 through 13 in the example)
- Two primary strategies discussed:
  - Replace missing values with the mean when the column distribution has no outliers.
  - Replace missing values with the median when the column distribution has outliers.
- Practical note: the instructor referred to a dataset (house_prices.csv) where numerical columns run from index 1 up to index 13 (index 0 is date). The implementation example targets that range (iloc starting from index 1 through index 13, i.e., index 1 to 13 inclusive; the discussion described selecting from index 1 until index 14 but not including 14 — effectively ending at index 13).
- Implementation hint: use DataFrame.iloc to select specific column index range and DataFrame.fillna(...) to replace nulls with mean/median.

### 3. Strategies for Categorical Data
- Primary/common strategy: replace missing categorical values with the mode (most frequently occurring value).
- Edge case: if multiple values tie for mode, handling depends on the algorithm/codebase (examples: alphabetic precedence or first-occurrence preference). Instructor noted this is implementation-dependent and can be "shady" in imperfect situations.

### 4. Implementation Example Details
- The instructor walked through code-level behavior: identify columns by type, then apply fillna on target columns.
- Example context: house_prices.csv — date is at index 0, numeric columns extend to index 13, categorical/object/text columns begin at index 14 onward.
- The process described: check for missing values -> apply mean/median replacement for numeric columns -> apply mode replacement for categorical columns -> verify results.

### 5. Duplicates and Unique Identifier
- Duplicate detection/removal should be based on the dataset's unique identifier (primary key), not on arbitrary columns like name, salary, city, etc.
- If the dataset lacks a unique identifier, create one (add an ID column) before proceeding. Instructor stressed this is critical and a sign to step back and wrangle data if missing.
- Typical operation: drop_duplicates(...) keyed to the unique identifier column.
- The instructor stressed: do not perform deduplication based on non-unique attributes; always use a unique identifier.

### 6. Standardization
- Standardize formats required for downstream algorithms — example given: converting a date column to a standard datetime format using to_datetime (or equivalent).
- Standardization increases compatibility with downstream feature engineering and modeling steps.

### 7. Process Sequencing and Scope
- The instructor repeatedly emphasized the sequence: identification -> cleaning (missing values, duplicates, standardization) -> (later) data transformation/feature engineering.
- Data cleaning was the immediate focus. Data transformation (normalization, scaling, aggregation, concatenation) would be covered after the break.

## Instructor Guidance / Opinions Highlighted
- Always check for missing values before handling them.
- Choose mean vs. median for numeric replacement based on presence/absence of outliers.
- For categorical replacement use mode; be aware of tie-handling differences between implementations.
- Never perform duplicate-removal based on arbitrary columns; use a unique identifier only.
- If no unique identifier exists, add one; lack of an identifier is a signal to pause and fix data-wrangling issues.
- Standardization of data formats (e.g., dates) is essential prior to downstream operations.

## Actionable Takeaways (explicit)
- Identify numeric vs. categorical columns early in the pipeline.
- For numeric columns:
  - Check for outliers. If none, replace nulls with mean; if outliers exist, use median.
- For categorical columns: replace nulls with mode; inspect ties and understand how your chosen library/function handles them.
- Ensure a unique identifier column exists; if not, create a deterministic ID column before deduplication.
- Convert date columns to standardized datetime types (e.g., pandas.to_datetime) to ensure downstream compatibility.
- Use DataFrame.iloc to select specific column ranges and DataFrame.fillna to implement replacements; use drop_duplicates keyed to the unique ID to remove duplicate rows.
