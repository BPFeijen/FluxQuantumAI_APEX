3. Loop sobre DEMO (roboforex) + LIVE (hantec): `executor.get_open_positions()` → `close_position(ticket)`.

5. Se `_total_positions_found > 0`:
   - Build canonical payload com `event_type="NEWS_EXIT"`, `action_type="CLOSE_ALL"`, `direction_affected="BOTH"`, `execution_state` baseado em `_total_closed_fail`.
   - Write `decision_live.json` via tmp+replace (atomic).
   - Append a `decision_log.jsonl`.
   - **`tg.notify_decision()`** — notifier lê `decision_live.json`, detecta `action=="PM_EVENT"` + `event_type=="NEWS_EXIT"`, envia mensagem com icon `📰` (icon map da Fase 2 M6.3).
6. Barbara vê no Telegram: **`📰 NEWS_EXIT — BOTH`** com detalhes do close agregado.

### Robustez
- Try/except wrapping previne que falha de emissão do PM_EVENT quebre o close flow principal (posições já fecharam antes).
- Se `_total_positions_found == 0` (EXIT_ALL chamado mas 0 posições abertas), nenhum PM_EVENT emitido — evita notificações spam sem semântica.
