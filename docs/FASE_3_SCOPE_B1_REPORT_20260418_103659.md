## Event flow after Fase 3 Scope B.1

Quando uma posição APEX fecha externamente (TP/SL/manual):

1. **Tick N:** `_run_checks` corre, `get_open_positions()` retorna lista com N posições. `_last_position_count = N`.
2. **Tick N+1 (entre ticks, TP é atingido):** MT5 fecha a leg automaticamente.
3. **Tick N+1:** `_run_checks` corre. `get_open_positions()` retorna N-1 posições.
4. **Condição `current_count < _last_position_count` é TRUE.**
5. `_history_watcher.find_closed_since_last_check()` chama `mt5.history_deals_get(T-2s, now)`.
6. Retorna 1 deal (o que fechou a leg). Label = `"TP_HIT"`.
7. `is_close_event(deal)` → True (TP_HIT está em CLOSE_LABELS).
8. `_handle_closed_deal(deal)`:
   - Direction = LONG ou SHORT (baseado no deal type)
   - `_find_trade_by_ticket(position_id)` → trade_rec
   - Se `ticket == leg1` → `event_type = "TP1_HIT"`
   - Se `ticket == leg2` ou `leg3` → `event_type = "TP2_HIT"`
   - `_emit_position_event(event_type=...)` → `_publish_canonical_pm_event()` → escreve decision_live.json + **Fase 2 notifica Telegram**.
9. Barbara vê no Telegram: `✅ TP1_HIT — LONG` ou `🏆 TP2_HIT — LONG` (icon map da Fase 2 Mudança 6.3).
